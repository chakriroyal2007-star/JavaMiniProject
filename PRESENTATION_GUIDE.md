Welcome to the official presentation guide for **EduPulse**. This document is designed to help you (a 2nd-semester CSE student) explain your project with confidence and answer any technical questions from your examiners.

## 🎭 The Presentation Script (The "Story")

### 1. Introduction
"Good morning everyone. Our project is called **EduPulse**. It is a gamified classroom management system. We created this to solve the problem of low engagement in online learning. Instead of just a boring list of files, we turned the classroom into an interactive experience where students earn XP and level up."

### 2. The Problem & Solution
"The main problem we identified is that students often feel unmotivated in traditional digital portals. Our solution, **EduPulse**, uses gamification. We have a Teacher Dashboard to manage resources and quizzes, and a Student Portal where every activity is rewarded with points and titles."

### 3. Key Features
"We have implemented:
- **Real-time Discussion Hub:** For group and personal chat with file sharing.
- **Interactive Quizzes:** With time limits, password protection, and instant result celebrations using the Confetti API.
- **Gamification Engine:** A custom logic that tracks levels and XP progress.
- **Resource Management:** Easy document sharing between teachers and students."

### 4. Technical Architecture
"We used a modern stack:
- **Backend:** Java 25 with Spring Boot 4.0.
- **Database:** H2 (In-memory/File-based) with JPA for data persistence.
- **Frontend:** Responsive UI using HTML, CSS, and JavaScript.
- **Security:** Spring Security for role-based access (Teacher vs Student)."

## ❓ Viva Questions & Answers (Q&A)

### Q1: Why did you choose Spring Boot for this project?
**A:** "Spring Boot allows us to create production-ready applications quickly. It provides 'Auto-configuration' and an embedded server (Tomcat), which means we don't have to set up a separate web server manually."

### Q2: What is JPA and how are you using it?
**A:** "JPA stands for **Java Persistence API**. It is a specification for managing relational data in Java. We use **Spring Data JPA** (Hibernate) to map our Java classes (like `User` or `Quiz`) to database tables automatically. This is called ORM (Object-Relational Mapping)."

### Q3: How does the Real-time Chat work? Are you using WebSockets?
**A:** "Currently, we use **HTTP Polling**. Every 3 seconds, the browser sends a small request to the server to check for new messages. This is a reliable and simple way to implement real-time features for a mini-project without the complexity of WebSockets."

### Q4: Can you explain a core OOP concept used in your code?
**A:** "We use **Encapsulation**. All our entity fields (like `userName` or `quizTitle`) are private, and we use Getters and Setters (generated by **Lombok**) to access them. This protects our data from being modified directly from outside the class."

### Q5: What happens to the data when you stop the server?
**A:** "We are using an **H2 Database** in 'file mode'. This means the data is saved in a local file on the disk (in the `/db` folder). So, even if the server stops, the data is persisted and will be there when we restart."

### Q6: What is 'Dependency Injection' in Spring?
**A:** "It is a design pattern where the Spring container 'injects' the objects needed by a class (like a Service or Repository) automatically. We use the `@Autowired` annotation to tell Spring to provide the required instance."

### Q7: How did you implement the XP/Leveling system?
**A:** "In the `QuizService`, we have logic that calculates the score after a submission. If a student passes, we update their XP in the `User` table. The frontend then calculates the level using a simple formula: `Math.floor(totalXP / 1000) + 1`."

## 🛠️ Technical Terms to Remember
- **Maven/Gradle:** Build tools used to manage dependencies (libraries).
- **Lombok:** A library that reduces boilerplate code like getters, setters, and constructors.
- **Controller:** The layer that handles incoming web requests.
- **Service:** The layer where the 'business logic' resides.
- **Repository:** The layer that communicates with the database.
